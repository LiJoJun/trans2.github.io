Templater

简介

Templater 是一种模板语言，允许你在笔记中插入变量和函数的结果。它还允许你执行JavaScript代码来操作这些变量和函数。

通过 Templater，你将能够创建强大的模板来自动化手动任务。

快速示例

以下是使用 Templater 语法的模板文件：

---
creation date: <% tp.file.creation_date() %>
modification date: <% tp.file.last_modified_date("dddd Do MMMM YYYY HH:mm:ss") %>
---

<< [[<% tp.date.now("YYYY-MM-DD", -1) %>]] | [[<% tp.date.now("YYYY-MM-DD", 1) %>]] >>

# <% tp.file.title %>

<% tp.web.daily_quote() %>

插入后将产生以下结果：

---
creation date: 2021-01-07 17:20
modification date: Thursday 7th January 2021 17:20:43
---

<< [[2021-04-08]] | [[2021-04-10]] >>

安装

你可以在 Obsidian 的“社区插件”选项卡中安装此插件。搜索“Templater”。

安装完 Templater 后，建议重启 Obsidian 应用，以确保一切运行正常。

术语

为了理解 Templater 的工作原理, 我们先第一几个术语

  • 一个包含 commands 的文件 模板 。
  • 以开始标签 <% 开头、以结束标签 %> 结尾的文本片段，我们将称之为 命令。
  • 函数 是一种对象，它可以在 命令 中被我们调用，并返回一个值(用于替换的字符串)

你可以使用的函数分为两种类型:

  • 内部函数。它们是插件内部内置的 预定义 函数。例如， tp.date.now 是一个内部函数，它将返回当前日期。
  • 用户函数。用户可以定义自己的函数。这些函数可以是系统命令 或 用户脚本。

示例

以下模板包含2个命令，调用2个不同的内部函数:

Yesterday: <% tp.date.yesterday("YYYY-MM-DD") %>
Tomorrow: <% tp.date.tomorrow("YYYY-MM-DD") %>

我们将在下一部分看到编写一些命令所需的语法。

语法

Templater 使用了一个自定义模板引擎 (rusty_engine) 的语法来声明命令。你可能需要一点时间来适应，但只要掌握了要领，这个语法其实并不难。

Templater 的所有函数都是可通过命令来调用的 JavaScript 对象。

命令语法

一条命令 必须 同时包含开始标签 <% 和结束标签 %>。

使用 tp.date.now 内部函数的完整命令如下: <% tp.date.now() %>

函数语法

对象层次结构

Templater 中的所有函数，无论是内部函数还是用户函数，都可以在tp对象下访问。可以说，我们所有的函数都是tp对象的"子项"。对象的"子项"。要访问一个对象的“子项”，我们必须使用点表示法:.

这意味着对 Templater 函数的调用总是以tp.<something>开头。

函数调用

要调用一个函数，我们需要使用函数调用的特定语法：在函数名后加一个左括号和一个右括号。

例如，我们会使用tp.date.now()来调用tp.date.now这个内部函数。

函数可以有参数和可选参数。它们位于左括号和右括号之间，如下所示:

tp.date.now(arg1_value, arg2_value, arg3_value, ...)

所有参数必须按正确顺序传入。

函数的参数有不同的类型。以下是函数可能类型的一个不完整列表:

  • string 表示一个字符串类型，参数值必须用单引号或双引号括起来 ("value"或'value')
  • number 表示一个整数类型，参数值必须是整数 (如15, -5等)
  • boolean 该表示一个布尔类型，参数值必须是 true 或 false （全部小写），除此之外别无其他。

调用函数时必须遵守参数的类型规则，否则函数将无法正常运行。

函数文档语法

Templater 内部函数的文档采用以下语法:

tp.<my_function>(arg1_name: type, arg2_name?: type, arg3_name: type = <default_value>, arg4_name: type1|type2, ...)

其中:

  • arg_name 表示该参数的 符号 名称，用于标识和理解参数的具体含义。
  • type 表示函数要求的参数类型。调用内部函数时必须遵守该类型的规则，否则会引发错误。

如果某个参数是可选的，则会在其后添加一个问号?，例如arg2_name?: type

如果一个参数有默认值，则使用等号=指定类型，例如arg3_name: type = <default_value>

如果一个参数可以有不同的类型，则使用竖线|进行分隔，例如arg4_name: type1|type2

语法警告

请注意，此语法仅用于文档说明，以便理解该函数要求接收哪些参数。

调用函数时，不得指定参数的名称、类型或默认值。只需提供如上所述参数的值即可。

示例

我们以 tp.date.now 这个内部函数的文档为例:

tp.date.now(format?: string = "YYYY-MM-DD", offset?: number|string, reference?: string, reference_format?: string)

这个内部函数有4个可选参数:

  • 类型为string的format，默认值为"YYYY-MM-DD"。
  • 类型为number或string的offset。
  • 类型为string的reference。
  • 类型为string的reference_format。

这意味着该内部函数的有效调用可以是:

  • <% tp.date.now() %>
  • <% tp.date.now("YYYY-MM-DD", 7) %>
  • <% tp.date.now("YYYY-MM-DD", 7, "2021-04-09", "YYYY-MM-DD") %>
  • <% tp.date.now("dddd, MMMM Do YYYY", 0, tp.file.title, "YYYY-MM-DD") %> *假设文件名的格式是 string :"YYYY-MM-DD"

另一方面，无效调用是指:

  • tp.date.now(format: string = "YYYY-MM-DD")
  • tp.date.now(format: string = "YYYY-MM-DD", offset?: 0)

设置

常规设置

  • Template folder location模板文件夹路径: 此路径下文件夹中的文件将作为可用的模板。
  • Syntax Highlighting on Desktop桌面端的语法高亮: 在编辑模式下为Templater命令添加语法高亮。
  • Syntax Highlighting on Mobile移动端的语法高亮: 在移动端编辑模式下为Templater命令添加语法高亮。请谨慎使用：这可能会导致移动平台上的实时预览无法正常显示。
  • Automatic jump to cursor自动跳转到光标位置: 会在插入模板后触发tp.file.cursor。还可以设置快捷键来手动触发tp.file.cursor
  • Trigger Templater on new file creation在新文件创建时触发模板生成器: Templater会监听新文件创建事件，如果文件内容符合您设定的规则，它就会替换新文件中找到的所有命令。这使得Templater能与“每日笔记核心插件”、“日历插件”、“回顾插件”、“笔记重构插件”等其他插件兼容。
    • 请务必在下方“文件夹模板”或“文件正则表达式模板”下设置规则。
    • 警告:如果在创建新文件时，文件内容未知或不安全，可能会带来风险。请确保每个新文件在创建时的内容都是安全的。

Template 快捷键

Template快捷键功能允许您将模板绑定到快捷键上。

文件夹模板

注意: 此设置默认处于隐藏状态。要查看它，请先开启Trigger Template on new file creation在新文件创建时触发模板。由于该设置与“文件正则表达式模板”互斥，启用其中一个将自动禁用另一个。

通过Folder Templates文件夹模板功能，你可以给文件夹指定一个模板，所选文件夹及其子文件夹将自动应用该模板。系统将采用匹配层级最深的规则，因此规则的排列顺序无关紧要。

如果需要添加一个通配规则，请对/根路径添加一条规则。

文件正则表达式模板

注意: 此设置默认处于隐藏状态。要查看该设置，请先启用Trigger Template on new file creation在新文件创建时触发模板。由于该设置与“文件夹模板”互斥，启用其中一项将自动禁用另一项。

您可以指定正则表达式，用于检测新文件的路径是否符合这些规则。如果正则表达式匹配，系统将自动使用相应的模板。规则将从上到下依次检测，并采用第一个匹配项。

如果需要添加一个通配规则，规则请以.*结尾。

使用Regex101之类的工具来验证您的正则表达式。

tp.date.now(format: string = "YYYY-MM-DD", offset?: number | string, reference?: string, reference_format?: string)

返回日期。

参数

  • format: 按照此参数的格式，设定输出日期的格式。默认为: "YYYY-MM-DD"详见Momentjs 格式参考。

  • offset: 日期偏移量(可以是正数或负数)，输入的数字将以天为单位加(负数等效为减)在参考日期上。 你还可以输入按照 ISO 8601 格式输入字符串。

  • reference: 参考日期，默认为今天日期。你也可以把文件名作为参考日期。

  • reference_format: 参考日期的格式。详见 Momentjs 格式参考。

示例
// 按照默认格式返回今天的日期
<% tp.date.now() %>
// 按照设定的格式返回今天的日期
<% tp.date.now("Do MMMM YYYY") %>
// 按照设定的格式返回一周前(七天前)的日期
<% tp.date.now("YYYY-MM-DD", -7) %>
// 按照设定的格式返回一周后(七天后)的日期
<% tp.date.now("YYYY-MM-DD", 7) %>
// 按照设定的格式返回一个月前的日期
<% tp.date.now("YYYY-MM-DD", "P-1M") %>
// 按照设定的格式返回一年后的日期
<% tp.date.now("YYYY-MM-DD", "P1Y") %>
// 按照设定的格式返回文件名为参考日期一天后的日期
<% tp.date.now("YYYY-MM-DD", 1, tp.file.title, "YYYY-MM-DD") %>
// 按照设定的格式返回文件名为参考日期一天前的日期
<% tp.date.now("YYYY-MM-DD", -1, tp.file.title, "YYYY-MM-DD") %>

tp.date.tomorrow(format: string = "YYYY-MM-DD")

只能返回明天的日期。

参数
  • format: 日期的输出格式。默认为: "YYYY-MM-DD"。详见Momentjs 格式参考文档。
示例
// Date tomorrow
<% tp.date.tomorrow() %>
// Date tomorrow with format
<% tp.date.tomorrow("Do MMMM YYYY") %>

tp.date.weekday(format: string = "YYYY-MM-DD", weekday: number, reference?: string, reference_format?: string)

返回特定的日期。

参数
  • format: 日期的输出格式。默认为"YYYY-MM-DD"。
  • weekday: 工作日的数字(0-6，0表示星期日)。默认为1(星期一)。
  • reference: 参考日期，找到参考日期所在的周，根据 weekday 确定是星期几的日期。默认为今天。
  • reference_format: 参考日期的输入格式。默认为null。
示例
// 按设定格式输出本周星期一的日期
<% tp.date.weekday("YYYY-MM-DD", 1) %>
// 按设定格式输出下周星期天的日期
<% tp.date.weekday("YYYY-MM-DD", 7) %>
// 按设定格式输出参考日期所在周的星期一
<% tp.date.weekday("YYYY-MM-DD", 1, "2021-04-09", "YYYY-MM-DD") %>
// 按格式输出参考日期所在周的上周天的日期
<% tp.date.weekday("YYYY-MM-DD", -7, "2021-04-09", "YYYY-MM-DD") %>

tp.date.yesterday(format: string = "YYYY-MM-DD")

只能返回昨天的日期。

参数
  • format: 日期的输出格式。默认为: "YYYY-MM-DD"。详见Momentjs 格式参考文档。
示例
// 基本用法
<% tp.date.yesterday() %>

// 自定义格式
<% tp.date.yesterday("Do MMMM YYYY") %>

Moment.js

Templater 允许你访问 moment对象，并使用该对象的所有功能。

详见 moment.js 文档.

示例
// 按设定格式输出文件名(如果文件以日期命名)
<% moment(tp.file.title, "YYYY-MM-DD").format("YYYY-MM-DD") %>
// 按设定格式输出当月的第一天的日期
<% moment().startOf("month").format("YYYY-MM-DD") %>
// 按设定格式输出当月的最后一天的日期
<% moment().endOf("month").format("YYYY-MM-DD") %>
// 按设定格式输出下个月的第一天的日期
<% moment().add(1, "month").format("YYYY-MM-DD") %>

示例

// 从参考日期开始的星期一

// 从参考日期开始的星期天
<% tp.date.weekday("YYYY-MM-DD", 0, "2021-04-09", "YYYY-MM-DD") %>

// 基本用法
<% tp.date.yesterday() %>

// 自定义格式
<% tp.date.yesterday("Do MMMM YYYY") %>

// 基本用法
<% tp.date.tomorrow() %>

// 自定义格式

tp.file.content

tp.file.create_new(template: TFile | string, filename: string = 'Untitled', open_new: boolean = false, folder?: TFolder | string)

使用指定的模板创建一个新文件。

Arguments
  • template: 要使用的模板路径（相对于模板文件夹）。
  • filename: 新文件的文件名。
  • open_new: 创建后是否打开新文件。默认为true。
  • folder: 存放新文件的目标文件夹（相对于保险库根目录）。可以使用"PATH/TO/FOLDERNAME" 或 tp.app.vault.getAbstractFileByPath("PATH/TO/FOLDERNAME")指定。默认为当前文件所在文件夹。

Examples
// File creation
<%* await tp.file.create_new("MyFileContent", "MyFilename") %>
// File creation with template
<%* await tp.file.create_new(tp.file.find_tfile("MyTemplate"), "MyFilename") %>
// File creation and open created note
<%* await tp.file.create_new("MyFileContent", "MyFilename", true) %>
// File creation in current folder
<%* await tp.file.create_new("MyFileContent", "MyFilename", false, tp.file.folder(true)) %>
// File creation in specified folder with string path
<%* await tp.file.create_new("MyFileContent", "MyFilename", false, "Path/To/MyFolder") %>
// File creation in specified folder with TFolder
<%* await tp.file.create_new("MyFileContent", "MyFilename", false, tp.app.vault.getAbstractFileByPath("MyFolder")) %>
// File creation and append link to current note
[[<% (await tp.file.create_new("MyFileContent", "MyFilename")).basename %>]]

在插入模板后，将光标定位到此处。

贡献

您可以通过开发新的内部函数/变量来为Templater做出贡献。

开发新功能的过程非常简单。

请记住，只有相关的提交才会被接受，不要提交只有您自己会使用的非常特定的内部变量/函数。

布局

内部变量/函数按模块分类。每个模块在src/InternalTemplates下都有专门的文件夹。

让我们以date模块为例。

它包含一个InternalModuleDate文件，其中定义并注册了所有与日期相关的内部变量和函数：

export class InternalModuleDate extends InternalModule {
    name = "date";

    async createStaticTemplates() {
        this.static_templates.set("now", this.generate_now());
        this.static_templates.set("tomorrow", this.generate_tomorrow());
        this.static_templates.set("yesterday", this.generate_yesterday());
    }

    async updateTemplates() {}

    generate_now() {
        return (format: string = "YYYY-MM-DD", offset?: number, reference?: string, reference_format?: string) => {
            if (reference && !window.moment(reference, reference_format).isValid()) {
                throw new Error("Invalid title date format, try specifying one with the argument 'reference'");
            }
            return get_date_string(format, offset, reference, reference_format);
        }
    }

    generate_tomorrow() {
        return (format: string = "YYYY-MM-DD") => {
            return get_date_string(format, 1);
        }
    }

    generate_yesterday() {
        return (format: string = "YYYY-MM-DD") => {
            return get_date_string(format, -1);
        }
    }
}

每个模块都继承自InternalModule抽象类，这意味着它们包含以下属性和方法：

  • this.app属性：Obsidian API的App对象。
  • this.file属性：模板将被插入的目标文件。
  • this.plugin属性：Templater插件对象。
  • this.static_templates属性：包含所有静态(名称；变量/函数)的映射。静态变量/函数意味着它在执行时不依赖于文件。每次插入新模板时，这类变量/函数不会更新，以节省开销。
  • this.dynamic_templates属性：与static_templates相同，只是它包含执行时依赖于文件的变量/函数。
  • this.createStaticTemplates()方法：注册该模块的所有静态内置变量/函数。
  • this.updateTemplates()方法：注册该模块的所有动态内置变量/函数。

如果需要，您可以在新的内置变量/函数中使用这些属性。

注册新的内部变量/函数

以下是注册模块中新内置变量/函数的不同步骤。

第一步:在模块内创建一个名为generate_<内置变量或函数名>()的方法，该方法将生成您的内置变量/函数，这意味着它将返回lambda函数（表示内置函数）或直接返回您想要暴露的内置变量。

所有生成方法按照内置变量/函数名称的字母顺序排序。

尝试为您的变量/函数提供一个好的、自解释的名称。

第二步: 根据您的内置变量/函数在执行时是否依赖于文件，在static_templates或dynamic_templates映射中注册它。注册发生在createStaticTemplates或updateTemplates中。

要注册您的变量/函数，请使用您之前定义的this.generate_<内置变量或函数名>()方法：

this.static_templates.set(<内置变量或函数名>, this.generate_<内置变量或函数名>());
或
this.dynamic_templates.set(<内置变量或函数名>, this.generate_<内置变量或函数名>());

内置变量/函数注册也按变量/函数名称的字母顺序排序。

第三步:将您的内置变量/函数文档添加到Templater的文档中。

完成了！感谢您为Templater做出贡献！

  • this.static_templates属性：包含所有静态(名称；变量/函数)的映射。静态变量/函数意味着它在执行时不依赖于文件。每次插入新模板时，这类变量/函数不会更新，以节省开销。
  • this.dynamic_templates属性：与static_templates相同，只是它包含执行时依赖于文件的变量/函数。
  • this.createStaticTemplates()方法：注册该模块的所有静态内置变量/函数。
  • this.updateTemplates()方法：注册该模块的所有动态内部变量/函数。

如果需要，您可以在新的内置变量/函数中使用这些属性。

注册新的内部变量/函数

以下是注册模块中新内置变量/函数的不同步骤。

第一步:在模块内创建一个名为generate_<内置变量或函数名>()的方法，该方法将生成您的内置变量/函数，这意味着它将返回lambda函数（表示内置函数）或直接返回您想要暴露的内置变量。

所有生成方法按照内置变量/函数名称的字母顺序排序。

尝试为您的变量/函数提供一个好的、自解释的名称。

第二步: 根据您的内置变量/函数在执行时是否依赖于文件，在static_templates或dynamic_templates映射中注册它。注册发生在createStaticTemplates或updateTemplates中。

要注册您的变量/函数，请使用您之前定义的this.generate_<内置变量或函数名>()方法：

this.static_templates.set(<内置变量或函数名>, this.generate_<内置变量或函数名>());
或
this.dynamic_templates.set(<内置变量或函数名>, this.generate_<内置变量或函数名>());

内部变量/函数注册也按变量/函数名称的字母顺序排序。

第三步:将您的内置变量/函数文档添加到Templater的文档中。

完成了！感谢您为Templater做出贡献！

用户函数

您可以在Templater中定义自己的函数。

您可以使用两种类型的用户函数：

  • 用户脚本函数
  • 用户系统命令函数

调用用户函数

您可以使用常规的函数调用语法调用用户函数：tp.user.<用户函数名>()，其中<用户函数名>是您定义的函数名。

例如，如果您定义了一个名为echo的用户系统命令函数，完整的命令调用将如下所示：

<% tp.user.echo() %>

无移动端支持

目前，用户函数在Obsidian移动版上不可用。

用户脚本函数

这种类型的用户函数允许您从JavaScript文件中调用JavaScript函数并获取其输出结果。

要使用脚本用户函数，您需要在Templater的设置中指定一个脚本文件夹。该文件夹必须能在您的保管库找到。

定义脚本用户函数

假设您在Templater的设置中将Scripts文件夹指定为脚本文件夹。

Templater将加载Scripts文件夹中的所有JavaScript（.js文件）脚本。

然后您可以创建名为Scripts/my_script.js的脚本（需要.js扩展名）。您可能需要在Obsidian外部创建该文件，因为Obsidian只创建markdown(.md)文件。

然后您将能够将您的脚本作为用户函数调用。函数名是对应的脚本文件名。

脚本应遵循CommonJS模块规范，并导出单个函数。

function my_function (msg) {
    return `来自我的脚本的消息: ${msg}`;
}
module.exports = my_function;

在此示例中，完整的命令调用如下所示：

<% tp.user.my_script("Hello World!") %>

这将输出来自我的脚本的消息: Hello World!。

全局命名空间

在用户脚本函数中，您仍然可以访问全局命名空间变量，如app或moment。

但是，您不能访问模板引擎作用域变量，如tp或tR。如果您想使用它们，必须将它们作为参数传递给您的函数。

函数参数

您可以根据函数的定义方式向函数传递任意数量的参数。

例如，您可以将tp对象传递给您的函数，以便能够使用Templater的所有内部变量/函数：<% tp.user.<用户函数名>(tp) %>

用户脚本文档

您可以选择使用TSDoc标准在方法文件的顶部记录脚本的功能。如果使用该功能，这将为您的用户脚本提供类似intellisense(智能感知)的体验，类似于其他templater函数的体验。

带文档的用户脚本示例

/**
 * 这会做一些很酷的事情
 */
function doSomething() {
    console.log('完成了某事')
}

module.exports = doSomething;

用户系统命令函数

这种类型的用户函数允许您执行系统命令并获取其输出结果。

系统命令用户函数需要在Templater的设置中启用。

定义用户系统命令函数

要定义新的用户系统命令函数，您需要定义一个函数名，并与系统命令关联。

为此，请转到插件设置并点击Add User Function添加用户函数。

完成后，Templater将创建一个以您定义的名称命名的用户函数，该函数将执行您的系统命令并返回其输出。

与内部函数一样，用户函数在tp JavaScript对象下可用，更具体地说是在tp.user对象下。

函数参数

您可以向用户函数传递可选参数。它们必须作为单个JavaScript对象传递，该对象包含属性及其对应的值：{arg1: value1, arg2: value2, ...}。

这些参数将以环境变量的形式提供给您的程序/脚本。

在我们之前的示例中，这将生成以下命令声明：<% tp.user.echo({a: "值 1", b: "值 2"})>。

如果我们的系统命令调用bash脚本，我们将能够使用$a和$b访问变量a和b。

系统命令中的内部函数

您可以在系统命令中使用内部函数。这些内部函数将在系统命令执行前被替换。

例如，如果您配置了系统命令cat <% tp.file.path() %>，在系统命令执行前，该表达式将被替换为cat /path/to/file。

命令

命令类型

Templater 定义了两种开始标签，它们定义了两种命令：

  • <%: 插值命令。它将输出其中表达式的结果。
  • <%*: JavaScript执行命令。它将执行其中的JavaScript代码。默认情况下不输出任何内容。

两种命令的结束标签都使用：%>

命令工具

除了不同类型的命令外，您还可以使用命令工具。它们也在命令的开始标签中声明。所有命令工具适用于所有命令类型。可用的命令工具有：

  • 空白控制
  • 动态命令

动态命令

通过这个命令工具，您可以将命令声明为"动态的"，这意味着此命令将在进入预览模式时解析。

要声明动态命令，请在命令开始标签后添加加号+如：<%+

这样，您的命令现在将只在预览模式下执行。

这对于像tp.file.last_modified_date这样的内置函数很有用，例如：

最后修改日期：<%+ tp.file.last_modified_date() %>

注意：动态命令存在已知问题，并且可能不会在未来继续维护（有关更多详细信息，请参阅此问题）。在大多数情况下，建议使用Dataview插件作为替代方案。

刷新问题

预览模式的一个“缺点”是，为了加快速度，它会将渲染后的笔记保存在缓存中。

这意味着您的动态命令将只在您打开笔记时渲染一次，但之后不会刷新。

如果您想刷新它，必须关闭笔记以清除缓存，然后再次打开它。

JavaScript执行命令

这种类型的命令允许我们执行JavaScript代码。

通过JavaScript执行命令，我们几乎可以做JavaScript允许我们做的所有事情。下面给出了一些示例。

我们仍然可以从这种类型的命令中访问tp对象和所有内部变量/函数。

JavaScript执行命令允许您访问全局命名空间变量。这意味着您可以访问诸如app或moment之类的内容。

异步函数

某些内部函数是异步的。在JavaScript执行命令中调用此类函数时，如有必要，不要忘记使用await关键字。

如何从JavaScript执行命令输出值？

有时，您可能希望在使用JS执行命令时输出某些内容。

当我们的模板引擎使用所有命令结果生成替换字符串时，该字符串会被存储在名为tR的变量中。这个字符串将包含处理后的文件内容。您可以从JS执行命令访问该变量。

这意味着，要从JS执行命令输出内容，您只需要将要输出的内容附加到那个tR字符串变量中。

例如，以下命令：<%* tR += "测试" %>将输出测试。

您还可以覆盖 tR 变量，以忽略模板引擎截至该点生成的所有内容。如果您希望在模板中包含前置信息或其他内容，但不希望在应用模板时插入这些内容，此方法会很有用。

例如，以下模板:

---
type: template
---
This is a person template.

<%* tR = "" -%>
---
type: person
---
# <% tp.file.cursor() %>

将会输出:

---
type: person
---
#

建议器和提示

重要的是要注意tp.system.prompt()和tp.system.suggester()都需要await语句才能将值保存到变量中

以下是使用JavaScript执行命令可以做的一些事情的示例：

<%* if (tp.file.title.startsWith("Hello")) { %>
这是一个hello文件！
<%* } else { %>
这是一个普通文件！
<%* } %>

这是一个seedling文件！
<%* } else { %>
这是一个普通文件！
<%* } %>

这是一个待办事项文件！
<%* } else { %>
这是一个已完成文件！
<%* } %>

function log(msg) {
    console.log(msg);

<%* tR += tp.file.content.replace(/stuff/, "things"); %>

空白控制

默认情况下，Templater中的命令不会删除任何换行符。命令被替换为它们的值，仅此而已。

有时候在插入命令后控制一些空白(空格)非常有用，此命令工具正是为此而设计的。

让我们看一个例子。以下模板：

<%* if (tp.file.title == "MyFile" ) { %>
这是我的文件！
<%* } else { %>
这不是我的文件！
<%* } %>
一些内容...

如果条件为假（当条件为真时也一样），将产生以下输出，注意空行：


这不是我的文件！

一些内容...

您可能想要删除由执行命令产生的空行，这些命令不会产生任何输出。

空白控制有一种特定的语法：

  • 开始标签后跟一个下划线_（<%_）将删除命令之前的所有空格
  • 结束标签前添一个下划线_（_%>）将删除命令之后的所有空格
  • 开始标签后跟一个短横杠(减号)-（<%-）将删除命令之前的一个换行符
  • 结束标签前添一个短横杠(减号)-（-%>）将修剪命令之后的一个换行符

在我们的例子中，要修复我们的模板以删除空行，我们将使用以下模板（注意标签末尾的减号-），以删除执行命令之后的空行：

<%* if (tp.file.title == "MyFile" ) { -%>
这是我的文件！
<%* } else { -%>
这不是我的文件！
<%* } -%>
一些内容...

这将产生以下输出：

这不是我的文件！
一些内容...
