Templater

简介

Templater 是一种模板语言，允许你在笔记中插入变量和函数的结果。它还允许你执行JavaScript代码来操作这些变量和函数。

通过 Templater，你将能够创建强大的模板来自动化手动任务。

快速示例

以下是使用 Templater 语法的模板文件：

---
creation date: <% tp.file.creation_date() %>
modification date: <% tp.file.last_modified_date("dddd Do MMMM YYYY HH:mm:ss") %>
---

<< [[<% tp.date.now("YYYY-MM-DD", -1) %>]] | [[<% tp.date.now("YYYY-MM-DD", 1) %>]] >>

# <% tp.file.title %>

<% tp.web.daily_quote() %>

插入后将产生以下结果：

---
creation date: 2021-01-07 17:20
modification date: Thursday 7th January 2021 17:20:43
---

<< [[2021-04-08]] | [[2021-04-10]] >>

术语

为了理解 Templater 的工作原理, 我们先第一几个术语

  • 一个包含 commands 的文件 模板 。
  • 以开始标签 <% 开头、以结束标签 %> 结尾的文本片段，我们将称之为 命令。
  • 函数 是一种对象，它可以在 命令 中被我们调用，并返回一个值(用于替换的字符串)

你可以使用的函数分为两种类型:

  • 内部函数。它们是插件内部内置的 预定义 函数。例如， tp.date.now 是一个内部函数，它将返回当前日期。
  • 用户函数。用户可以定义自己的函数。这些函数可以是系统命令 或 用户脚本。

示例

以下模板包含2个命令，调用2个不同的内部函数:

Yesterday: <% tp.date.yesterday("YYYY-MM-DD") %>
Tomorrow: <% tp.date.tomorrow("YYYY-MM-DD") %>

我们将在下一部分看到编写一些命令所需的语法。

在Templater中，我们使用以下术语：

  • 命令：一个由开始标签<%和结束标签%>包围的代码块，例如<% tp.date.now() %>。命令是用户与Templater交互的基本方式。

  • 内置函数：Templater预定义的函数，用于执行特定任务，如tp.date.now()、tp.file.title等。这些函数按功能分组到不同的模块中。

  • 用户函数：用户自定义的函数，可以是JavaScript脚本或系统命令。

  • 变量：存储值的容器，如tp（Templater的主对象）或tR（模板结果）。

函数语法

对象层次结构

Templater 中的所有函数，无论是内部函数还是用户函数，都可以在tp对象下访问。可以说，我们所有的函数都是tp对象的"子项"。对象的"子项"。要访问一个对象的“子项”，我们必须使用点表示法:.

这意味着对 Templater 函数的调用总是以tp.<something>开头。

函数调用

要调用一个函数，我们需要使用函数调用的特定语法：在函数名后加一个左括号和一个右括号。

例如，我们会使用tp.date.now()来调用tp.date.now这个内部函数。

函数可以有参数和可选参数。它们位于左括号和右括号之间，如下所示:

tp.date.now(arg1_value, arg2_value, arg3_value, ...)

所有参数必须按正确顺序传入。

函数的参数有不同的类型。以下是函数可能类型的一个不完整列表:

  • string 表示一个字符串类型，参数值必须用单引号或双引号括起来 ("value"或'value')
  • number 表示一个整数类型，参数值必须是整数 (如15, -5等)
  • boolean 该表示一个布尔类型，参数值必须是 true 或 false （全部小写），除此之外别无其他。

调用函数时必须遵守参数的类型规则，否则函数将无法正常运行。

函数文档语法

Templater 内部函数的文档采用以下语法:

tp.<my_function>(arg1_name: type, arg2_name?: type, arg3_name: type = <default_value>, arg4_name: type1|type2, ...)

其中:

  • arg_name 表示该参数的 符号 名称，用于标识和理解参数的具体含义。
  • type 表示函数要求的参数类型。调用内部函数时必须遵守该类型的规则，否则会引发错误。

如果某个参数是可选的，则会在其后添加一个问号?，例如arg2_name?: type

如果一个参数有默认值，则使用等号=指定类型，例如arg3_name: type = <default_value>

如果一个参数可以有不同的类型，则使用竖线|进行分隔，例如arg4_name: type1|type2

语法警告

请注意，此语法仅用于文档说明，以便理解该函数要求接收哪些参数。

调用函数时，不得指定参数的名称、类型或默认值。只需提供如上所述参数的值即可。

示例

我们以 tp.date.now 这个内部函数的文档为例:

tp.date.now(format?: string = "YYYY-MM-DD", offset?: number|string, reference?: string, reference_format?: string)

这个内部函数有4个可选参数:

  • a format of type string, with a default value of "YYYY-MM-DD".
  • an offset of type number or of type string.
  • a reference of type string .
  • a reference_format of type string .

这意味着该内部函数的有效调用可以是:

  • <% tp.date.now() %>
  • <% tp.date.now("YYYY-MM-DD", 7) %>
  • <% tp.date.now("YYYY-MM-DD", 7, "2021-04-09", "YYYY-MM-DD") %>
  • <% tp.date.now("dddd, MMMM Do YYYY", 0, tp.file.title, "YYYY-MM-DD") %> *假设文件名的格式是 string :"YYYY-MM-DD"

另一方面，无效调用是指:

  • tp.date.now(format: string = "YYYY-MM-DD")
  • tp.date.now(format: string = "YYYY-MM-DD", offset?: 0)

tp.date.now(format: string = "YYYY-MM-DD", offset?: number | string, reference?: string, reference_format?: string)

返回日期。

参数

  • format: 按照此参数的格式，设定输出日期的格式。默认为: "YYYY-MM-DD"详见Momentjs 格式参考。

  • offset: 日期偏移量(可以是正数或负数)，输入的数字将以天为单位加(负数等效为减)在参考日期上。 你还可以输入按照 ISO 8601 格式输入字符串。

  • reference: 参考日期，默认为今天日期。你也可以把文件名作为参考日期。

  • reference_format: 参考日期的格式。详见 Momentjs 格式参考。

示例

<% tp.date.now("YYYY-MM-DD", 7) %>

<% tp.date.now("YYYY-MM-DD", "P-1M") %>
// 按照设定的格式返回一年后的日期
<% tp.date.now("YYYY-MM-DD", "P1Y") %>

让我们以date模块为例。

export class InternalModuleDate extends InternalModule {
    name = "date";

    async createStaticTemplates() {
        this.static_templates.set("now", this.generate_now());
        this.static_templates.set("tomorrow", this.generate_tomorrow());
        this.static_templates.set("yesterday", this.generate_yesterday());
    }

    async updateTemplates() {}

    generate_now() {
        return (format: string = "YYYY-MM-DD", offset?: number, reference?: string, reference_format?: string) => {
            if (reference && !window.moment(reference, reference_format).isValid()) {
                throw new Error("Invalid title date format, try specifying one with the argument 'reference'");
            }
            return get_date_string(format, offset, reference, reference_format);
        }
    }

    generate_tomorrow() {
        return (format: string = "YYYY-MM-DD") => {
            return get_date_string(format, 1);
        }
    }

    generate_yesterday() {
        return (format: string = "YYYY-MM-DD") => {
            return get_date_string(format, -1);
        }
    }
}

每个模块都继承自InternalModule抽象类，这意味着它们包含以下属性和方法：

  • this.app属性：Obsidian API的App对象。
  • this.file属性：模板将被插入的目标文件。
  • this.plugin属性：Templater插件对象。
  • this.static_templates属性：包含所有静态(名称；变量/函数)的映射。静态变量/函数意味着它在执行时不依赖于文件。每次插入新模板时，这类变量/函数不会更新，以节省开销。
  • this.dynamic_templates属性：与static_templates相同，只是它包含执行时依赖于文件的变量/函数。
  • this.createStaticTemplates()方法：注册该模块的所有静态内置变量/函数。
  • this.updateTemplates()方法：注册该模块的所有动态内置变量/函数。

如果需要，您可以在新的内置变量/函数中使用这些属性。

定义脚本用户函数

}
module.exports = my_function;

就是这样，您的命令现在将只在预览模式下执行。

这对于像tp.file.last_modified_date这样的内置函数很有用，例如：

最后修改日期：<%+ tp.file.last_modified_date() %>

注意：动态命令存在已知问题，并且可能不会在未来继续维护（有关更多详细信息，请参阅此问题）。在大多数情况下，建议使用Dataview插件作为替代方案。

刷新问题

预览模式的一个"缺点"是它将渲染后的笔记放入缓存中，以加快速度。

这意味着您的动态命令将只在您打开笔记时渲染一次，但之后不会刷新。

如果您想刷新它，必须关闭笔记以清除缓存，然后再次打开它。

空白控制

默认情况下，Templater中的命令不会删除任何换行符。命令被替换为它们的值，仅此而已。

有时候在插入命令后控制一些空白(空格)非常有用，此命令工具正是为此而设计的。

让我们看一个例子。以下模板：

<%* if (tp.file.title == "MyFile" ) { %>
这是我的文件！
<%* } else { %>
这不是我的文件！
<%* } %>
一些内容...

如果条件为假（当条件为真时也一样），将产生以下输出，注意空行：


这不是我的文件！

一些内容...

您可能想要删除由执行命令产生的空行，这些命令不会产生任何输出。

空白控制有一种特定的语法：

  • 开始标签后跟一个下划线_（<%_）将删除命令之前的所有空格
  • 结束标签前添一个下划线_（_%>）将删除命令之后的所有空格
  • 开始标签后跟一个短横杠(减号)-（<%-）将删除命令之前的一个换行符
  • 结束标签前添一个短横杠(减号)-（-%>）将修剪命令之后的一个换行符

在我们的例子中，要修复我们的模板以删除空行，我们将使用以下模板（注意标签末尾的减号-），以删除执行命令之后的空行：

<%* if (tp.file.title == "MyFile" ) { -%>
这是我的文件！
<%* } else { -%>
这不是我的文件！
<%* } -%>
一些内容...

这将产生以下输出：

这不是我的文件！
